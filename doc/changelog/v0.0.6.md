# Changelog v0.0.6

## Logs

**DPS related**

- Changed True DPS calculation logic: previously calculated per entity, now based on global active combat time
- Added a setting to control whether global active combat time is displayed (DPS Meter > Theme > Real-time > Timer > Show Active Combat Time)
- When history exceeds 200 entries, older entries are now automatically deleted by time on the next app launch, and the sequence is reset
- Fixed inconsistent time display between history overview and detail pages; now uniformly using the difference between the first and last combat packets

**Buff related**

- Reorganized the settings interface into: Skill CD, Buff Monitor, Character Panel, Custom Panel, Enable Window
- Added global buff alias settings: search for a buff and rename it (e.g., rename "[Zeal]" to "Life Fluctuation"). (Real-time Monitor > Buff Monitor > Buff Alias Settings. This setting applies globally and is not affected by preset switching)
- Standalone mode: added "Category Quick Monitor" for monitoring all food or alchemy buffs at once with unified layout. Group mode: added "Food" and "Alchemy" shortcut buttons for one-click monitoring of all food/alchemy buffs. (Real-time Monitor > Buff Monitor > Category Quick Monitor)
- Added Custom Panel for counters and buff monitoring, displayed uniformly as progress bars. Supports settings for row spacing, font size, etc. Buffs you don't want to display with icons can be shown here instead.
- Changed the display format for buffs without icons: now displayed in a single line, with configurable row spacing, etc. Supports reverting to the old format
- Introduced a state machine linking buffs and damage types for counting special buffs triggered by specific damage accumulation rather than buff maintenance. Examples: Phantasm Impact count, Transcendence trigger count. (Real-time Monitor > Custom Panel > Add Counter)
- Buff duration format updated: added grouped and hourly display options
- Added support for setting a "Toggle Overlay Window" hotkey (DPS Meter > Settings > Hotkeys)

**Panel stats related**

- Added display for cooldown reduction and cooldown acceleration stats

**Notes**

- If you previously used 0.0.2/0.0.3, the first time you use 0.0.6 you need to delete `resonance-logs-cn.db` from the `%LOCALAPPDATA%\resonance-logs-cn` directory and restart the application
