# Changelog v0.0.5

## Logs

**Buff related**

- Standalone mode no longer displays unselected buffs; special buffs are hidden unless selected

**DPS related**

- Reset logic changes
  - Removed reset logic based on dungeon state, avoiding issues where the server sends dungeon state arbitrarily (e.g., guild hunt single boss death triggering a dungeon-end state)
  - Reset flag is now set with a 3-second async delay to prevent excess damage packets from generating multiple junk history entries
  - Added configuration to skip specific task IDs, avoiding mid-fight resets in multi-phase battles with multiple target IDs (e.g., robot raid sends multiple "boss eliminated" targets, and phase 2 switches to a decoding target)
  - Current reset logic: Master dungeons automatically split trash and boss into 2 history entries; raids split each boss independently; wipe auto-resets; for same-scenario resets, the reset triggers on the next attack (Master dungeon: after clearing trash, auto-reset upon attacking boss | Raid: auto-reset when attacking a new boss | After wipe: auto-reset on next boss attack)
- Refactored dungeon query logic; master dungeon difficulty is now appended
- History list page now shows player + class icon for each encounter
- Added support for displaying season power rating (DPS Meter > Settings > General Settings)

**Mod related**

- Player info is now written to the database immediately, preventing mods from failing to retrieve the latest data

**Character panel stats related**

- Added character panel stat monitoring to the skill monitor page (Skill Monitor > Character Panel Stats)

**Notes**

- If you previously used 0.0.2/0.0.3, the first time you use 0.0.5-beta you need to delete `resonance-logs-cn.db` from the `%LOCALAPPDATA%\resonance-logs-cn` directory and restart the application
